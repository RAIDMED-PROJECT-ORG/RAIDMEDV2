Use Testcontainers Cloud with GitHub Actions

1.Create a new service account

The Service Account's name is used to attribute CI usage. It must be unique.

This trial account lets you create one Service Account for your CI, to run a single concurrent session.

Learn WhyUpgrade
2.Set the TC_CLOUD_TOKEN environment variable

To use Testcontainers Cloud with your GitHub Actions you need to set TC_CLOUD_TOKEN to your token value generated in step 1. This can be achieved by setting the environment variable directly in your workflow by leveraging GitHub’s encrypted secrets.

3.Add Testcontainers Cloud to your workflow

Add a step to your GitHub Actions workflow that starts the Testcontainers Cloud agent before you run your tests. You can use the following script:

jobs:
    build:
        runs-on: ...
        steps:
            - uses: actions/checkout@v3

            # ... some other preparation steps (dependencies, compilation as a separate step, etc) ...

            - name: Setup Testcontainers Cloud Client
              uses: atomicjar/testcontainers-cloud-setup-action@v1
              with:
                  token: ${{ secrets.TC_CLOUD_TOKEN }}

            # ... existing steps executing your Testcontainers tests go here ...
